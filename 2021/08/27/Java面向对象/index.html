Java面向对象

本文最后更新于：2021年8月29日 早上

一、类和对象

  1. 对象

    万物皆对象，现实存在的客观事物，面向对象即关注事物的信息

  2. 类

    类是对象的模板，确定对象拥有的特征（属性）和行为（方法），对象是类的实例

  3. 创建类和对象

    实例化对象：Dog dog = new Dog();

  4. 构造方法

    public 构造方法名(参数) {

    ​ 初始化代码块

    }

    当没有指定构造方法时，系统会自动添加无参的构造方法；而当有了构造方法时，系统不会自动添加无参的构造方法。所以一般都会先写一个无参构造方法。

  5. this 关键字

    代表当前对象

    可通过 this()在构造方法中调用另一个构造方法

二、封装

  1. 什么是封装？

    将类的某些信息隐藏在类的内部，不允许外部程序直接访问，只能通过方法来对隐藏的信息进行访问和操作。简单来讲，就是隐藏对象的信息，留出访问的接口。

  2. 封装的实现

    • 修改属性的访问修饰符为 private
    • 创建 getter/setter 方法
    • 在getter/setter 方法加入属性的控制语句

  3. static 关键字

    • 修饰变量：静态成员，属于类，对象共享；类加载时产生，销毁时释放，生命周期长。（不能加在局部变量前）
    • 修饰方法：静态方法，静态方法不能直接访问同一个类非静态成员
    • this 和 super 不能在静态方法中使用

  4. 代码块

    • 普通代码块：顺序执行
    • 构造代码块（在类中书写）：创建对象时调用，优先于构造方法执行
    • 静态代码块：在代码块前加上 static 关键字，只执行一次

三、继承

  1. 什么是继承？

    继承是一种类与类的关系，子类获取父类的所有属性和方法的机制。

  2. 继承的实现

    • 编写父类

    • 编写子类，继承父类

      子类无法访问父类私有成员，父类不可以访问子类特有成员

  3. 方法的重写

    • 在子类中定义，方法名，参数类型、顺序、个数都相同
    • 返回值和访问修饰符允许有变化，有一定条件：返回值为引用类型，访问修饰符不允许缩小

  4. super 关键字

    • super. : 父类对象的引用
    • super() : 调用父类构造方法，必须写在第一行

  • 继承后的初始化顺序：父类静态成员 -> 子类静态成员 -> 父类对象构造 -> 子类构造对象

  • 访问修饰符

  1. Object 类
    • boolean equals(Object obj): 判断两个对象的引用是否一致
    • String toString(): 打印类的字符串表现形式
  2. final 关键字
    • 修饰类：防止继承
    • 修饰方法：防止重写（不能修饰构造方法）
    • 修饰变量：
      • 方法内的局部变量：在使用之前进初始化赋值
      • 类成员变量：只能在定义时、构造方法、构造代码块中进行初始化
      • 修饰基本数据类型后值不允许修改，修饰引用变量后不能再指向另一个对象

四、多态

  1. 什么是多态？

    同一操作作用于不同的对象，可以有不同的解释，产生不同的执行结果。

  2. 向上转型

    父类引用指向子类实例

  3. 向下转型（强制转换）

    子类引用指向父类对象，可以调用子类特有的方法

  4. instanceof 运算符

  5. 抽象

    • 抽象类：不允许实例化，可以通过向上转型，指向子类实例
    • 抽象方法：不允许包含方法体，子类需要重写父类的抽象方法

  6. 接口

    • 接口访问修饰符：public 和 默认
    • 接口里的方法默认为抽象方法，常量加上了 public final
